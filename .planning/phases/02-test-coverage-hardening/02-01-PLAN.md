The `inheritedJobPodSpec` helper handles inheritance for deployment-level CronJobs using `hasKey` + `ternary` for most fields. Test the following fields with the seven-category pattern. For each field, write 4 test cases (the categories that demonstrate distinct behavior): **default** (not set anywhere), **inheritance** (set on deployment, not on job), **override** (set on both, job wins), **explicit-empty** (set to `{}` or `[]` on job, blocks inheritance).

Skip "enabled" (always enabled in these tests), "disabled" (tested implicitly by existing enabled:false tests), and "failure" (covered by Task 1 negative tests).

If Phase 1 regression tests already cover a category for a field, add a comment referencing the existing test name instead of duplicating.

**Fields to test (6 fields, ~24 test cases):**

1. **nodeSelector** (inherited via `ternary $job.nodeSelector $deploy.nodeSelector (hasKey $job "nodeSelector")` -- rendered by `inheritedJobPodSpec`)
   - Path: `spec.jobTemplate.spec.template.spec.nodeSelector`
   - default: isNull
   - inheritance: deploy has `nodeSelector: {tier: backend}`, cronjob does not -- assert `nodeSelector.tier` = `backend`
   - override: deploy has `tier: backend`, cronjob has `tier: worker` -- assert `tier` = `worker`
   - explicit-empty: cronjob has `nodeSelector: {}` -- assert isNull

2. **tolerations** (inherited via helper)
   - Path: `spec.jobTemplate.spec.template.spec.tolerations`
   - default: isNull
   - inheritance: deploy has `tolerations: [{key: "gpu", operator: "Exists", effect: "NoSchedule"}]`
   - override: both have tolerations, cronjob value wins
   - explicit-empty: cronjob has `tolerations: []` -- assert isNull

3. **affinity** (inherited via helper -- uses `ternary $job.affinity $deploy.affinity (hasKey $job "affinity")` pattern)
   - Path: `spec.jobTemplate.spec.template.spec.affinity`
   - Same 4 categories

4. **podSecurityContext** (inherited via `$podSecCtx := ternary $job.podSecurityContext $deploy.podSecurityContext (hasKey $job "podSecurityContext")`)
   - Path: `spec.jobTemplate.spec.template.spec.securityContext`
   - default: isNull
   - inheritance: deploy has `podSecurityContext: {runAsUser: 1000}` -- assert present
   - override: cronjob has `podSecurityContext: {runAsUser: 2000}` -- assert runAsUser=2000
   - explicit-empty: cronjob has `podSecurityContext: {}` -- assert isNull

5. **securityContext** (container-level, inherited via `$secCtx := ternary $job.securityContext $deploy.securityContext (hasKey $job "securityContext")`)
   - Path: `spec.jobTemplate.spec.template.spec.containers[0].securityContext`

6. **imagePullSecrets** (3-level chain: job > deploy > global, via `hasKey`)

   - default: isNull (no global either)
   - inheritance: deploy has `imagePullSecrets: [{name: "my-secret"}]`, cronjob does not
   - override: both have it, cronjob wins
   - explicit-empty: cronjob has `imagePullSecrets: []` -- assert isNull (blocks deploy AND global)
