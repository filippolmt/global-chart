- HIGH: `false`, `0`, or `""` is a valid user-intended value that would be silently replaced

- LOW/SAFE: nil-guard pattern (`default (dict)`), enum defaults, or string concatenation helpers

Present HIGH and MEDIUM findings in a table:
| # | File | Line | Current Code | Issue | Proposed Fix |
The "Proposed Fix" column must contain the exact replacement code snippet (hasKey + ternary or hasKey + if/else).

Specific findings to include (from research, verify line numbers against actual source):
- cronjob.yaml line 47: `default true $job.automountServiceAccountToken` -- HIGH, fix with hasKey
- cronjob.yaml lines 63-64: `default 2 $job.successfulJobsHistoryLimit` and `failedJobsHistoryLimit` -- MEDIUM, 0 is valid K8s
- cronjob.yaml lines 257-259: same defaults on deployment-level cronjob section
- hook.yaml lines 65, 81: `default "5"` and `default "10"` on `$command.weight` -- MEDIUM, weight "0" is valid
- hook.yaml lines 265, 286: same defaults on deployment-level hook section
- deployment.yaml line 20: `$deploy.replicaCount | default 2` -- MEDIUM, 0 replicas may be valid

Also include a summary count of LOW/SAFE `default` calls (approximately 30+) noting they need no changes.

**Section 3 -- TMPL-03: Truncation Guard Analysis**
