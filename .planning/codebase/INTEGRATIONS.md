# External Integrations

**Analysis Date:** 2026-03-15

## APIs & External Services

**External Secrets Operator:**
- ExternalSecret CRD (`external-secrets.io/v1`) — syncs secrets from external secret stores into Kubernetes Secrets
  - SDK/Client: Kubernetes CRD (no SDK — templated via `charts/global-chart/templates/externalsecret.yaml`)
  - Auth: Configured via `secretStoreRef` in values (points to a `SecretStore` or `ClusterSecretStore` CR on the cluster)
  - Required fields per secret: `remote.key`, `secretkey`, `secretstore.kind`, `secretstore.name`
  - Cluster-side prerequisite: External Secrets Operator must be installed independently

**CodeRabbit:**
- AI code review on pull requests
  - Config: `coderabbit.yaml` at repo root
  - Integration type: GitHub App (no secrets required in this repo)
  - Enabled tools: yamllint, gitleaks, checkov, actionlint, github-checks, languagetool

**Renovate Bot:**
- Automated dependency update PRs
  - Config: `renovate.json` at repo root
  - Preset: `config:recommended`
  - No secrets required in this repo

## CI/CD & Deployment

**Hosting:**
- GitHub Pages (chart repository) — chart releases published via `helm/chart-releaser-action`
- Source: `https://github.com/filippomerante/global-chart`

**CI Pipeline:**
- GitHub Actions
  - CI workflow: `.github/workflows/helm-ci.yml` — triggers on push to `main` and pull requests
    - Step 1: `make lint-chart` — `helm lint --strict` against all 16 test scenarios
    - Step 2: `make unit-test` — helm-unittest via Docker (220 tests, 16 suites)
    - Step 3: `make generate-templates` — renders all scenarios to `generated-manifests/`
    - Step 4: Uploads `generated-manifests/` as GitHub Actions artifact
  - Release workflow: `.github/workflows/release.yml` — triggers on push to `main`
    - Uses `helm/chart-releaser-action@v1.7.0` with `skip_existing: true`
    - Authenticates with `CR_TOKEN: ${{ secrets.GITHUB_TOKEN }}` (standard GitHub token, no custom secret)

## Container Registries

**Default registry behavior:**
- `global.imageRegistry` (empty string by default) — optionally prepends a shared registry prefix to all images
- Smart registry detection in `global-chart.imageString` helper: if the first path segment of an image contains `.`, `:`, or equals `localhost`, it is treated as a registry and the global prefix is skipped
- `global.imagePullSecrets` — fallback pull secrets for all deployments/jobs when not set at component level
