## Directory Purposes

**`charts/global-chart/templates/`:**
- Purpose: All Kubernetes manifest templates rendered by Helm
- Contains: One `.yaml` file per resource kind, plus `_helpers.tpl` for shared logic
- Key files: `_helpers.tpl` (all helpers), `deployment.yaml` (primary resource), `cronjob.yaml` and `hook.yaml` (two-part templates with root-level and deployment-scoped sections)

**`charts/global-chart/tests/`:**
- Purpose: helm-unittest test suites, one per template file
- Contains: YAML files using `helm.sh/unittest` schema with `suite`, `templates`, and `tests` keys
- Key files: `deployment_test.yaml`, `helpers_test.yaml`, `hook_test.yaml`, `cronjob_test.yaml`

**`tests/`:**
- Purpose: Values files for `helm lint --strict` scenarios and `helm template` output generation
- Contains: Named scenario value files, organized flat or in subdirectories (only `test01/` is a subdirectory)
- Key files: `test01/values.01.yaml` (comprehensive), `multi-deployment.yaml`, `deployment-hooks-cronjobs.yaml`

**`generated-manifests/`:**
- Purpose: Rendered Kubernetes manifests output for visual inspection and artifact upload
- Generated: Yes (by `make generate-templates`)
- Committed: No (gitignored, uploaded as CI artifact)

**`baseline-manifests/`:**
- Purpose: Reference manifests for regression comparison
- Generated: No (manually maintained)
- Committed: Yes

## Key File Locations

**Entry Points:**
- `charts/global-chart/values.yaml`: All configurable values with defaults
- `charts/global-chart/Chart.yaml`: Chart metadata, version, Kubernetes requirements

**Configuration:**
- `Makefile`: All developer workflow commands
- `.github/workflows/helm-ci.yml`: CI pipeline definition
- `.kube-linter-config.yaml`: Static analysis rules for generated manifests
- `renovate.json`: Automated dependency update config

**Core Logic:**
- `charts/global-chart/templates/_helpers.tpl`: All naming, labeling, image resolution, and shared rendering helpers
- `charts/global-chart/templates/deployment.yaml`: Primary workload template
- `charts/global-chart/templates/hook.yaml`: Most complex template (two-part: root-level + deployment-scoped hooks with full SA lifecycle)
- `charts/global-chart/templates/cronjob.yaml`: CronJob template (two-part, same structure as hook.yaml)

**Testing:**
- `charts/global-chart/tests/deployment_test.yaml`: Deployment template assertions
- `charts/global-chart/tests/helpers_test.yaml`: Helper function unit tests
- `tests/test01/values.01.yaml`: Comprehensive integration scenario

## Naming Conventions

**Template files:**
- Lowercase with hyphens matching the Kubernetes resource kind: `deployment.yaml`, `networkpolicy.yaml`, `mounted-configmap.yaml`
- Helpers file prefixed with underscore: `_helpers.tpl`
- Test files match template name with `_test` suffix: `deployment_test.yaml`

**Helper names:**
- Prefixed `global-chart.` then camelCase: `global-chart.deploymentFullname`, `global-chart.renderImagePullSecrets`
- Shared renderers prefixed `render`: `renderVolume`, `renderDnsConfig`, `renderResources`, `renderImagePullSecrets`
- Boolean helpers return string `"true"`/`"false"` (not boolean): `global-chart.deploymentEnabled`

**Values keys:**
- camelCase at all levels: `replicaCount`, `imagePullPolicy`, `serviceAccount.automount`, `autoscaling.targetCPUUtilizationPercentage`
- Maps use camelCase keys: `deployments.myApp`, `cronJobs.cleanup`, `hooks.post-upgrade.migrate`
- Hook type keys match Helm hook annotation values: `pre-install`, `post-upgrade`, etc.

**Test scenario files:**
- Snake-case or hyphenated slugs: `multi-deployment.yaml`, `deployment-hooks-cronjobs.yaml`, `hooks-sa-inheritance.yaml`

## Where to Add New Code

**New Kubernetes resource type (e.g., a new template):**
- Template: `charts/global-chart/templates/{kind}.yaml`
- Test suite: `charts/global-chart/tests/{kind}_test.yaml`
- Values documentation: add to `charts/global-chart/values.yaml` with `# -- ` helm-docs annotation
- Test scenario: add a `tests/{scenario}.yaml` and register in `Makefile` `TEST_CASES`

**New helper function:**
- Location: `charts/global-chart/templates/_helpers.tpl`
- Prefix the define name with `global-chart.`
- Add a unit test in `charts/global-chart/tests/helpers_test.yaml`

**New per-deployment feature (e.g., new optional sub-resource):**
- Add iteration in a new template file following the `range $name, $deploy := .Values.deployments` + `deploymentEnabled` guard pattern
- Use `global-chart.deploymentFullname` and `global-chart.deploymentLabels` for naming and labels
- Document the new field in `charts/global-chart/values.yaml` under the deployments example block

**New test scenario:**
- Create `tests/{scenario-slug}.yaml`
- Register in `Makefile` at `TEST_CASES` with format `tests/{file}:{namespace}:{slug}`

**Utilities / shared rendering logic:**
- All shared helpers: `charts/global-chart/templates/_helpers.tpl`
- Shared renderers follow the pattern: accept a single argument, return empty string when nil/empty, use `{{- with . -}}` trim syntax to avoid leading newlines
